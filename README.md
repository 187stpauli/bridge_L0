# Stargate Bridge

Асинхронный скрипт для быстрого бриджа токенов (ETH/USDC) через протокол Stargate между сетями Arbitrum и Base.

## Возможности

- Поддержка бриджа ETH и USDC в режиме fast
- Асинхронная архитектура для эффективной работы
- Поддержка прокси и безопасное хранение ключей
- Автоматическая проверка балансов и расчет комиссий
- Отслеживание статуса транзакции в целевой сети
- Подробное логгирование процесса бриджа
- Детальная валидация параметров перед выполнением операций

## Требования

- Python 3.9+
- Web3.py
- asyncio
- eth-abi
- dotenv

## Установка

1. Клонируйте репозиторий:
```bash
git clone <репозиторий>
cd bridge_L0
```

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

3. Создайте файл `.env` в корневой директории:
```
PRIVATE_KEYS={"my_wallet_key":"ВАШ_ПРИВАТНЫЙ_КЛЮЧ"}
PROXIES={"my_proxy":"login:pass@host:port"}
```

## Конфигурация

Настройте параметры бриджа в файле `config/settings.json`:

```json
{
  "proxy": "ENV:my_proxy",
  "private_key": "ENV:my_wallet_key",
  "from_network": "Base",
  "to_network": "Arbitrum",
  "token": "ETH",
  "amount": 0.001
}
```

### Доступные параметры:

- `proxy` - HTTP прокси в формате (login:pass@host:port) или ENV:ключ для загрузки из .env
- `private_key` - Приватный ключ кошелька или ENV:ключ для загрузки из .env
- `from_network` - Сеть-отправитель (поддерживаются: Arbitrum, Base)
- `to_network` - Сеть-получатель (поддерживаются: Arbitrum, Base) 
- `token` - Токен для бриджа (ETH или USDC)
- `amount` - Количество токенов для отправки (минимум 0.0001)

## Использование

```bash
python main.py
```

## Структура проекта

- `abi/` - ABI контрактов для взаимодействия с блокчейном
- `client/` - Клиентский код для работы с Web3
- `config/` - Настройки и валидация конфигурации
- `constants/` - Константы и данные сетей
- `modules/` - Основные модули скрипта
- `utils/` - Утилиты для работы скрипта

## Поток выполнения

1. Загрузка и валидация конфигурации
2. Инициализация Web3 клиента
3. Проверка балансов и расчет комиссии
4. Выполнение approve (для ERC-20 токенов)
5. Отправка транзакции бриджа
6. Ожидание подтверждения транзакции
7. Отслеживание получения токенов в целевой сети

## Расширение функциональности

Для добавления новых сетей необходимо:

1. Добавить информацию о сети в `constants/networks_data.json`
2. Обновить валидацию в `config/configvalidator.py`

## Безопасность

- Приватные ключи хранятся только в .env файле (не включен в систему контроля версий)
- Валидация параметров перед выполнением операций
- Проверка балансов для предотвращения ошибок из-за недостатка средств
